	컨테이너
		격리된 공간에서 프로세스가 동작하는 기술
		cpu, 메모리의 손실이 없음 
		cpu, 메모리 사용량을 제한할 수 있음
		독립적으로 실행되어 Virtual Machine을 사용하는 느낌
		컨테이너에 접속하여 명령어를 입력할 수 있음
		컨테이너 생성 시간이 빠름(1~2초)
	
	이미지
		컨테이너 실행에 필요한 파일과 설정
		상태값을 가지지 않고 변하지 않음
		컨테이너는 이미지를 실행한 상태
		추가 또는 변하는 값은 컨테이너에 저장
		같은 이미지로 여러 개의 컨테이너를 만들수 있음
		컨테이너의 상태에 영향을 받지 않음
		가상머신에 비해 작은 용량
		Docker Hub를 통해 버전 관리 및 배포 가능
		이미지 명 Dockerfile 
		
		Dockerfile
			DSL(Domain Specific Language) 언어로 작성된 이미지 파일
			단순 텍스트 파일 
			사용할 수 있는 키워드 20개
			From과 Run으로 이미지를 만들 수 있음
				From : 베이스 이미지 지정
				Run : 명령어를 실행
			
	도커 설치하기
		리눅스 컨테이너 기술 mac, windows에서는 가상머신에 설치가 됨
