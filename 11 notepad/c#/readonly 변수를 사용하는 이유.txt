	변수를 상수로 만들어버려, 변수값을 바꿀 수 없게 만드는 const와 유사해 보이지만
	Compile 시점에 값을 확정하지 않음
	
	!!!!readonly는 프로그램이 메모리에 올라가는 runtime 시점에 이르러서야 값을 확정하지
	
	readonly 초기화
		변수를 선언하는 경우
		생성자 안에서 변수값을 바꾸는 경우
		
	readonly는 (정의에 제한이 있지만) 일반 변수처럼 런타임 중에 값을 설정할 수 있기 때문에,
	compile 중에 처리되는 const와 같은 성능 상의 이점은 없음
	그렇기 때문에 const를 사용할 수 있는 곳이라면 const를 사용하는 것이 조금 더 성능에 기여할 수 있음

readonly
	읽기전용 변수, 초기화 후 수정할 수 없음
	비 정적 non static 변수와 유사하게 동작
	상수는 클래스에 대한 값, 반면 읽기 전용 변수는 객체에 대한 값

const 
	값이 변하지 않는 변수값을
	기본저긍로 Static 
	상수를 만드는 경우 반드시 값이 초기화 되어야 함
	정적 변수와 동일하게 동작, 객체를 생성할 필요가 없음
	정적 변수와 상수의 유일한 차이점은 정적 변수의 값은 수정할 수 있음
