
.NET 용어 
	https://docs.microsoft.com/ko-kr/dotnet/standard/glossary#aot
	
어셈블리
	애플리케이션이나 다른 어셈블리에서 호출할 수 있는 API 컬렉션을 포함할 수 있는 .dll 또는 .exe 파일
	어셈블리에는 인터페이스와 클래스, 구조체, 열거형, 대리자와 같은 형식이 포함될 수 있음
	프로젝트의 bin 폴더에 있는 어셈블리를 바이너리라고도 함

CLR 공용 언어 런타임
	메모리 할당 및 관리를 처리 
	앱을 실행할 뿐만 아니라 JIT컴파일러를 사용하여 즉시 코드를 생성하고 컴파일하는 가상 머신
	
크로스 플랫폼
	각 운영 체제에 맞게 다시 작성할 필요 없이 다양한 운영 체제에서 사용할 수 있는 애플리케이션을 개발하고 실행하는 기능
	다양한 플랫폼에서 애플리케이션 간에 코드를 다시 사용하고 일관성을 유지할 수 있음
	
프레임워크
	일반적으로 특정 기술을 기반으로 하는 애플리케이션의 개발 및 배포를 용이하게 하는 포괄적인 API 컬렉션을 
		예) ASP.NET Core, Windows Forms
	경우에 따라 프레임워크는 .NET 구현을 나타냄
	
GC 가비지 수집기
	자동 메모리 관리의 구현체 
	개체가 사용한 메모리에서 더 이상 사용되지 않는 메모리를 해제
	
IL 중간 언어
	C#과 같은 상위 수준 .NET언어가 IL이라는 하드웨어 독립 명령 집합으로 컴파일 됨
	IL은 MSIL(Microsoft IL) 또는 CIL(공통 IL) 이라고도 함
	
JIT Just-In-Time 컴파일러
	AOT와 유사, IL을 프로세서에서 이해하는 기계어 코드로 변환
	AOT와 달리 JIT 컴파일은 요청 시 수행되며 코드가 실행되어야 하는 것과 동일한 컴퓨터에서 수행
	
AOT Ahead-Of-Time 컴파일러
	JIT와 비슷, IL을 기계어 코드로 변환
	JIT 컴파일과는 달리 AOT 컴파일은 애플리케이션이 실행되기 전에 수행되며 일반적으로 다른 컴퓨터에서 수행
	
라이브러리
	앱이나 다른 라이브러리에서 호출할 수 있는 API 컬렉션
	
Mono
	작은 런타임이 필요할 떄 주로 사용되는 오픈 소스 플랫폼 간 .NET 구현
	주로 작은 사용 공간이 필요한 앱에 초점을 맞춤
	.NET Standard 버전을 모두 지원
	
.NET	
	일반적으로 .NET은 .NET Standard 및 모든 .NET 구현과 워크로드를 나타내는 포괄적인 용어
	구체적으로 말하면 .NET은 모든 신규 개발에 권장되는 .NET 구현, 즉 .NET 5(및 .NET Core) 이상 버전을 나타냄
	
.NET Framework
	Windows에서만 실행되는 .NET의 구현
	CLR(공용 언어 런타임), BCL(기본 클래스 라이브러리) 및 ASP.NET, Windows Forms, WPF 등의 애플리케이션 프레임워크 라이브러리를 포함
	
.NET SDK
	개발자가 .NET 5(및 .NET Core) 이상 버전용 .NET 애플리케이션과 라이브러리를 만드는 데 사용할 수 있는 라이브러리 및 도구 집합

NET Standard
각 .NET 구현에서 사용할 수 있는 .NET API의 공식 사양입니다.

.NET Standard 
	사양을 라이브러리라고도 합니다. 
	라이브러리는 API 구현체를 포함하므로 규격(인터페이스)뿐만 아니라 .NET Standard를 “라이브러리”라고 잘못 부르기도 합니다.
	
platform
	Windows와 macOS, Linux, iOS, Android 같은 운영 체제와 해당 운영 체제가 실행되는 하드웨어

런타임
	일반적으로 관리형 프로그램의 실행 환경
	
스택
	애플리케이션을 만들고 실행하는 데 사용되는 프로그래밍 기술 집합입니다.
	“.NET 스택”은 .NET Standard 및 모든 .NET 구현을 나타냅니다. “.NET 스택”이라는 문구는 하나의 .NET 구현체를 나타냅니다.

UWP
	유니버설 Windows 플랫폼
	IoT(사물 인터넷)용 터치 가능 Windows 애플리케이션 및 소프트웨어를 빌드하는 데 사용되는 .NET 구현	
	PC, 태블릿, 휴대폰, Xbox와 같은 대상으로 지정할 수 있는 다양한 종류의 디바이스를 통합하도록 설계
	중앙 집중식 앱 스토어, 실행 환경(AppContainer), Win32를 대체할 Windows API(WinRT) 등 많은 서비스를 제공
	앱은 C++과 C#, Visual Basic, JavaScript로 작성 (C#과 Visual Basic을 사용할 경우 .NET API는 .NET 5(및 .NET Core) 이상 버전에서 제공)
